# Быстрый старт

## Вариант 1: Автоматический запуск (рекомендуется)

```bash
./start.sh
```

## Вариант 2: Ручной запуск

### Шаг 1: Создайте .env файл (опционально)

Файл `.env` не обязателен, так как все значения имеют значения по умолчанию. Но для продакшена рекомендуется его создать:

```bash
cp ENV_EXAMPLE.txt .env
```

### Шаг 2: Запустите все сервисы

```bash
docker-compose up --build
```

Или если используете новый Docker Compose V2:

```bash
docker compose up --build
```

### Шаг 3: Дождитесь запуска всех сервисов

Вы увидите логи всех сервисов. Дождитесь сообщений о том, что все сервисы запущены.

### Шаг 4: Откройте приложение

После запуска откройте в браузере:

- **Frontend (веб-интерфейс)**: http://localhost:3000
- **API документация**: http://localhost:8000/docs
- **API напрямую**: http://localhost:8000
- **Через Nginx**: http://localhost:80

### Шаг 5: Войдите в систему

При первом запуске автоматически создается административный пользователь:

- **Email**: `admin@example.com`
- **Password**: `admin123`

## Остановка

Нажмите `Ctrl+C` в терминале, затем:

```bash
docker-compose down
```

Или для полной очистки (включая данные БД):

```bash
docker-compose down -v
```

## Проверка статуса

Проверьте, что все контейнеры запущены:

```bash
docker-compose ps
```

## Просмотр логов

```bash
# Все сервисы
docker-compose logs -f

# Конкретный сервис
docker-compose logs -f api
docker-compose logs -f worker
docker-compose logs -f frontend
```

## Решение проблем

### Порт уже занят

Если порты 3000, 8000 или 80 заняты, измените их в `docker-compose.yml`:

```yaml
ports:
  - "3001:3000"  # Вместо 3000:3000
```

### Ошибка при сборке

Попробуйте очистить кэш Docker:

```bash
docker-compose down
docker system prune -a
docker-compose up --build
```

### Worker не запускается

Проверьте, что Redis доступен:

```bash
docker-compose exec redis redis-cli ping
# Должно вернуть: PONG
```

### База данных не подключается

Проверьте логи PostgreSQL:

```bash
docker-compose logs postgres
```

## Первый скан

1. Войдите в систему через веб-интерфейс
2. Создайте проект (если нужно)
3. Создайте скан, указав домен или IP адрес
4. Дождитесь завершения сканирования (статус изменится на "done")
5. Просмотрите результаты и скачайте PDF отчет
